## Comparison: Old vs New

### OLD Search (Before)
```
1. User types query character by character
   - Each key = API call
   - "phÃ²ng" (5 chars) = 5 API calls
   
2. No search history/suggestions
   - User types full query manually
   
3. Filters applied separately
   - Each filter change = separate API call
   
4. No caching
   - Same search 2 minutes later = new API call

âŒ Result: 20+ API calls for one actual search
```

### NEW Search (After)
```
1. User types query with debounce
   - 800ms wait after last keystroke
   - Suggestions from cached popular/history
   
2. Popular searches & history shown
   - Often user just clicks a suggestion
   
3. Hybrid search combines query + filters
   - All changes in one API call
   
4. Multi-level caching
   - Browser cache (5 min)
   - localStorage history (forever)
   - Redis backend cache

âœ… Result: 1-2 API calls for complete search
```
